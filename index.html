todo-report

todo-report is a local, branch-aware CLI that measures TODO age, compares TODO drift across branches, lints TODO structure, and rolls those signals into a health summary.

Why teams use it

It gives visual teams a readable health story, Unix-heavy teammates a filter-friendly CLI, and repo owners a way to spot stale work, structural drift, and TODO hygiene problems without changing the source data.

Four working views of the same TODO system

The tool is built around three core features, with health acting as the summary layer on top.

Feature 1

age

Show how old top-level TODOs are by tracing first appearance in Git history.

Feature 2

drift

Compare TODO state across branches, including parent items, detail files, and subtasks.

Feature 3

lint

Catch malformed IDs, broken detail links, checked-parent issues, and orphaned TODO files.

Summary

health

Roll age, lint, and optional drift counts into one report for humans or other tools.

Report types

The tool covers single repos, monorepos, and fleets without changing command families.

Single repo

age drift lint health detect

Use these when one repo has one active queue or one explicit nested index.

Monorepo

indexes + health --all-indexes

Discover multiple authoritative TODO roots and summarize them together.

Fleet

fleet health

Roll repo lists or repeated --repo flags into one cross-repo summary.

Export

text markdown json tsv

Render for humans, scripts, or saved JSON snapshots through --write-json.

One tool, three scopes

The same command family works at the repo, monorepo, and fleet levels.

Single Repo

health

Use this for one queue rooted at TODO/TODO.md or an explicit --index path.

Monorepo

indexes

Discover multiple authoritative TODO roots and summarize them with health --all-indexes.

Fleet

fleet health

Roll many repos together from --repo-list or repeated --repo flags.

Compatibility

detect

Inspect a repo’s TODO dialect before adopting the tool or tightening lint expectations.

Supported TODO styles

The parser supports mixed real-world team styles rather than assuming one perfect format.

Example commands

The CLI stays readable by default, but it also exports markdown, json, and tsv, and it can narrow large runs with repo and index filters while returning script-friendly exit codes.

examples 
todo-report age --repo ~/lab/cswg/coordination --branch jj
todo-report drift --repo ~/lab/cswg/coordination --branch-a main --branch-b jj
todo-report lint --repo ~/lab/cswg/coordination --branch jj --format markdown
todo-report indexes --repo ~/lab/wire-lab --branch main --format text
todo-report indexes --repo ~/lab/wire-lab --branch main --include-index SIM-beta --format text
todo-report health --repo ~/lab/wire-lab --branch main --all-indexes --format text
todo-report health --repo ~/lab/cswg/coordination --branch jj --all-indexes --compare main --format text
todo-report fleet health --repo-list repos.txt --branch main --all-indexes --format text
todo-report fleet health --repo ~/lab/cswg/coordination --repo ~/lab/wire-lab --branch main --all-indexes --format text
todo-report fleet health --repo-list repos.txt --branch main --all-indexes --include-repo wire-lab --exclude-index retired/ --format text
todo-report drift --repo ~/lab/cswg/coordination --branch-a main --branch-b jj --format text; echo $?
todo-report fleet health --repo-list repos.txt --branch main --all-indexes --write-json fleet-health.json --format text
todo-report health --repo ~/lab/wire-lab --branch main --index protocols/wire-lab.d/TODO/TODO.md --format json
